## Выбранная предметная область

Интернет-магазин

Реализованный класс: `Product`

## Ход мыслительного процесса

1. **Выбор сущности**

Из предложенных сущностей (Product, CartItem, Customer, Order, Payment) выбрал `Product`, так как он является базовой единицей интернет-магазина и содержит достаточное количество характеристик для реализации требований лабы.

---

2. **Определение атрибутов**

Выделил ключевые свойства товара:

- `name` — название товара  
- `price` — цена  
- `discount` — скидка (%)  
- `stock` — количество на складе  
- `active` — доступность товара  

Чтобы пройти критерий на “минимум 4 атрибута” и добавить логическое состояние.

---

3. **Инкапсуляция**

Все атрибуты сделаны защищёнными (`_name`, `_price`, …), для

- ограничения прямого доступа
- управление изменением через свойства
- добавления валидации

---

4. **Валидация данных**

Валидация вынесена в отдельный файл `validate.py`,

Реализованы проверки:

- типа данных
- диапазонов значений
- логической корректности (например, пустое имя)

---

5. **Декораторы (`@property`)**

Используются для:

- безопасного доступа к данным
- добавления setter-методов с проверкой

---

6. **Атрибут класса**

Добавил `TAX_RATE`, общий для всех товаров.

---

7. **Бизнес-методы**

Реализованы:

- `price_after_discount()` — цена со скидкой  
- `price_with_tax()` — цена с налогом  
- `purchase()` — покупка товара  
- `restock()` — пополнение склада  

---

8. **Логическое состояние**

Добавлено состояние `active`:

- товар можно деактивировать (`deactivate`)
- товар нельзя купить, если он не активен

---

9. **Магические методы**

Реализованы:

- `__str__` — удобный вывод для пользователя  
- `__repr__` — техническое представление  
- `__eq__` — сравнение объектов  

---

## Ответы на вопросы (Практическое занятие №1)

1. **Что такое класс?**  
Класс — это пользовательский тип данных, который описывает структуру и поведение объектов.

2. **Что такое инкапсуляция?**  
Инкапсуляция — это сокрытие внутренней реализации объекта и предоставление доступа через контролируемый интерфейс.

3. **Разница между атрибутом класса и экземпляра?**  
Атрибут класса общий для всех объектов, атрибут экземпляра — уникален для каждого объекта.

4. **Зачем нужны свойства (`@property`)?**  
Позволяют контролировать доступ к данным и добавлять логику при чтении/изменении.

5. **Что такое магические методы?**  
Это специальные методы (`__str__`, `__eq__`, …), которые определяют поведение объекта при встроенных операциях.
