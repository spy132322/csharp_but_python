## Краткое описание процесса

Класс был выбран из предметной области интернет-магазина как базовая и наиболее понятная сущность — товар (`Product`). При разработке ориентировался на реальные характеристики товара: цена, скидка, наличие на складе и доступность. Сразу заложил инкапсуляцию через закрытые поля и добавил валидацию, чтобы объект нельзя было создать или изменить в некорректном состоянии. Логику проверок вынес отдельно для удобства и переиспользования. Далее добавил бизнес-методы, отражающие реальные действия (покупка, пополнение склада), и ввёл состояние товара (`active`), чтобы показать зависимость поведения от данных. В конце реализовал магические методы для удобной работы с объектом.

---

## Краткое описание класса

`Product` — это класс, представляющий товар в интернет-магазине.
Содержит поля:

- `name` — название товара  
- `price` — цена  
- `discount` — скидка в процентах  
- `stock` — количество на складе  
- `active` — статус доступности товара

## Методы класса

- `price_after_discount()` — возвращает цену с учётом скидки  
- `price_with_tax()` — возвращает цену с учётом налога  
- `purchase(quantity)` — выполняет покупку и уменьшает количество товара  
- `restock(amount)` — увеличивает количество товара  
- `activate()` — делает товар доступным  
- `deactivate()` — делает товар недоступным  

Магические методы:

- `__str__` — строковое представление для пользователя  
- `__repr__` — техническое представление объекта  
- `__eq__` — сравнение объектов
