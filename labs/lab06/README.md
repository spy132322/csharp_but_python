## 2. Описание реализованных типов и контейнеров

### Аннотации типов

Во всех классах из предыдущих лабораторных были добавлены:

- типы атрибутов

Пример:

```python
def calculate_price(self) -> float:

```

---

### Generic-класс `TypedCollection`

В файле `container.py` реализован универсальный контейнер:

```python
class TypedCollection(Generic[T])
```

Контейнер хранит элементы определённого типа и поддерживает:
- добавление элементов
- удаление элементов
- получение списка элементов
- поиск
- фильтрацию
- преобразование элементов

Используемые `TypeVar`:

```python
T = TypeVar("T")
R = TypeVar("R")
```

- `T` — тип элементов коллекции
- `R` — тип результата для метода `map`

---

### Методы контейнера

Реализованы методы:

```python
add()
remove()
get_all()
size()
clear()
contains()
find()
filter()
map()
```

---

### Protocol

Реализованы два протокола:

```python
class Displayable(Protocol):
    def display(self) -> str:
        ...
```

```python
class Scorable(Protocol):
    def score(self) -> float:
        ...
```

---

### Ограниченные TypeVar

```python
D = TypeVar("D", bound=Displayable)
S = TypeVar("S", bound=Scorable)
```

Это позволяет создавать контейнеры только для объектов,
которые поддерживают нужные методы.

---

### Структурная типизация

Классы:
- `Product`
- `LiquidProduct`
- `TangibleProduct`
- `ServiceProduct`

не наследуются от `Protocol`,
но подходят под них благодаря наличию методов:
- `display()`
- `score()`
